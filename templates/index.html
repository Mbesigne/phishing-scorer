🔐 Comprendre SPF / DKIM / DMARC
SPF (Sender Policy Framework)

Vérifie que l'IP de l'expéditeur est autorisée à envoyer pour le domaine. Un spf=fail signifie que l'email prétend venir d'un domaine mais est envoyé depuis une IP non autorisée — signe fort de spoofing.

DKIM (DomainKeys Identified Mail)

Signature cryptographique apposée sur l'email. Un dkim=fail signifie que la signature ne correspond pas — l'email a été modifié ou falsifié en transit.

DMARC (Domain-based Message Authentication)

Politique du domaine sur les échecs SPF/DKIM. dmarc=fail + p=reject devrait normalement faire rejeter l'email — s'il arrive quand même, c'est suspect.

🆘 Que faire si c'est du phishing ?
  1. Ne cliquez PAS sur les liens et ne téléchargez aucune pièce jointe.
  2. Ne répondez pas à l'email et ne fournissez aucun identifiant.
  3. Signalez sur signal-spam.fr (France) ou phishing-initiative.fr.
  4. Si vous avez cliqué : changez vos mots de passe immédiatement et activez le 2FA.
  5. Prévenez votre responsable sécurité ou votre hébergeur si nécessaire.
⚙️ Documentation API

Base URL : http://localhost:5000

POST /analyze

Analyse complète d'une URL avec email et headers optionnels.

curl -X POST http://localhost:5000/analyze \
  -H "Content-Type: application/json" \
  -d '{
    "url": "http://amaz0n-promo.tk/login",
    "email_text": "Urgent ! Votre compte est suspendu.",
    "headers": "From: noreply@amazon.com\nAuthentication-Results: dkim=fail"
  }'
GET /api/analyze?url=https://...

Analyse rapide via paramètre URL (pour intégrations externes).

curl "http://localhost:5000/api/analyze?url=https://paypal-secure.tk"
GET /history?limit=50&min_score=70

Retourne l'historique des analyses. Filtres : limit, min_score, max_score, domain.

curl "http://localhost:5000/history?limit=20&min_score=50"
POST /batch-analyze

Analyse un lot de 50 URLs maximum.

curl -X POST http://localhost:5000/batch-analyze \
  -H "Content-Type: application/json" \
  -d '{"urls": ["http://bit.ly/abc", "https://amazon.com", "http://paypa1.tk"]}'
GET /export/csv GET /export/json

Exporte l'historique complet.

// JavaScript — Exemple d'appel
const res = await fetch('/analyze', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ url: 'http://example.com' })
});
const data = await res.json();
console.log(data.score, data.risk_level, data.details);
DELETE /history/<id>

Supprime une analyse par son ID.

curl -X DELETE http://localhost:5000/history/42
